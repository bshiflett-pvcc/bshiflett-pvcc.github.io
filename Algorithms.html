Phases of computer program development:

Programming structures:

      Sequential structure:Top-down execution
      Decision structures: if/then/else
      Iterative (looping):
      • While loop: continue to loop as long as something is true
      • repeat/until loop: continue to loop until something is true
      • for loop: loop your given number
      Recursive structures:
          Repeat your set of instructions as a subtask of itself
          repetition continues until:
          • degenerative case. which is a termination condition, is met
          • degenerative case is also called a base case

Two clasic programming or database problems

      Sorting: Descending order or Ascending order
      • Sorting numbers
      • sorting strings in accordance with ASCII values
      Searching: find specific data that meet a certain criteria
          In database:
          • Filters: temporary search
          • Queries: search criteria are saved so they don't have to be recreated
      Two popular search techniques:
          Sequential search: search from top to bottom or bottom to top, one item at a time
          Binary search:
          • only works if data is sorted first
          • start in middle, then go to half of that, etc.

Two classic program goals:

      Efficiency: Program takes minimal time and/or resources
      Correctness: Program runs in expected ways and works for all expected conditions

Software testing and verification

      Kinds of tests:
      • Acceptance testing: Verifies that the whole system works like it's supposed to and nothing is wrong
      • Integration testing: Makes sure that that the software components are working together
      • Unit testing: Making sure that every software unit does is working properly
      • Functional testing: Evaluates the functionality of an application by testing its individual functions
      • Performance testing: Tests how the software operates from different types of workloads
      • Stress testing: Tests to see how much it takes to make it fail.
      • Usability testing: Tests how easy it is for customer use
      Software testing is done to detect:
      • Architectual flaws no defintion
      • poor design decisions no defintion
      • Invalid or incorrect functionality
      • Security vulnerability no defintion
      • Scalabilitiy issues no defintion
      Testing best practices:
      • Continous testing :enables software to be validated in realistic test environments earlier in the process improving design and reducing rissk
      • configuration management:centrally maintain test assets and track what software builds to test
      • Servic virtualization:simulates the services and systems that are missing or not yet completed, enabling teams to reduce dependencies and test sooner.
      • Defect or "bug tracking:allow teams to track defects, measure their scope and impact, and uncover related issues
      • Metrics and reporting:Reporting and analytics enable team members to share status, goals and test results.
